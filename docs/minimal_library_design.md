# crophg 最小库设计

## 目标

`crophg` 只保留论文最终叙事真正需要的章节接口，不继续承载旧仓库中大量并行试验、临时脚本和一次性输出。

## 公开与内部边界

### 内部分析

- `3.1A`: `H-only` 部署损失
- `3.1B`: `G` 的补偿与 `G+FULLH` 互补
- `3.2A`: 四场景下 `G+H` 的有效窗口
- `3.2B`: `H-only` 下单一 VI 的跨场景变化
- `3.2C`: `G` 背景下单一 VI 的真实增量与变化

这些章节作为论文内部支撑，不作为最终对外 API 暴露。

### 对外开放

- `3.3A`: 最终模型定义与输入表示
- `3.3B`: 最终模型预测表现与压缩收益
- `3.4A`: 生长推进曲线
- `3.4B`: `anchor×VI` 选择演化

这些章节是最小库后续需要保留的正式公开入口。

## 代码组织原则

- `src/crophg/public/`: 对外开放章节
- `src/crophg/internal/`: 内部章节
- `src/crophg/framework.py`: 章节注册表与论文映射
- `src/crophg/cli.py`: 统一命令行入口
- `scripts/`: 薄脚本入口，只负责调用包内模块

## 当前阶段

当前仅完成最小库骨架与章节边界定义。
具体实验逻辑将分批从旧仓库迁移，但迁移时只保留论文最终口径需要的最小实现。
